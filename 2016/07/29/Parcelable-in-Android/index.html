Parcelable in Android

发表于 2016-07-29   |  

在android中，各个component之间传输／分享数据的方式有很多种， 本文将介绍如何使用Parcelable传递复杂数据。

Basic

在Android中数据传输方式有这个几种：

  • Shared Preferences
  • Database
  • Static class
  • Intents

其中前两个对于数据需要持久化的情形。
由于Android程序一般运行在同一个process中，因此可以使用static class来分享数据。同时可以使用singleton pattern 来保证数据的独一性。

Intent

Intent 有三种方式传递数据：

  • Direct: put our data into intents directly
  • Bundle: create a bundle and set the data here
  • Parcelable: It is a way of “serializing” our object.

Parcelable Basic

Interface for classes whose instances can be written to and restored from a Parcel. Classes implementing the Parcelable interface must also have a non-null static field called CREATOR of a type that implements the Parcelable.Creator interface.

Parcelable是一个接口、用来实现序列化. 定义了将数据写入Parcel，和从Parcel中读出的接口。一个实体（用类来表示），如果需要封装到消息中去，就必须实现这一接口，实现了这一接口，该实体就成为“可打包的”了。

Serializable接口与Parcelable接口的区别

  • Serializable是JDK提供的接口，而Parcelable是Android SDK提供的。
  • Serializable序列化是基于磁盘的，而Parcelable是基于内存的。在内存中读写的效率要高于磁盘，所以Android中跨进程传递对象都是使用Parcelable。

实现 Parcelable

实现parcelable接口，需要我们实现下面几个方法：

  • describeContents(): 内容接口描述，默认返回0就可以
  • writeToParcel(Parcel dest, int flags): 方法将类的数据写入外部提供的Parcel中.即打包需要传递的数据到Parcel容器保存，以便从parcel容器获取数据
  • 静态的Parcelable.Creator接口，本接口有两个方法：
    • createFromParcel(Parcel in) 从Parcel容器中读取传递数据值，封装成Parcelable对象返回逻辑层
    • newArray(int size) 创建一个类型为T，长度为size的数组，仅一句话（return new T[size])即可。方法是供外部类反序列化本类数组使用
